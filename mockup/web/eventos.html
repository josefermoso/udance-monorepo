🔧 Consideraciones Técnicas - Lista de Eventos y Filtros

  • Modelo de Datos: La página consulta la tabla events.
  • Filtros: Los filtros de la barra lateral se construyen dinámicamente poblando los checkboxes desde las tablas de taxonomía:
    • "Estilo de Baile" se llena con datos de la tabla dance_styles.
    • "Categoría de Evento" se llena con datos de la tabla event_categories.
  • API Endpoint: La búsqueda se realiza a través de un único endpoint que acepta parámetros de consulta.
    Ejemplo: GET /api/events?dance_style_id=2&category_id=2&city=Palermo
  • La API debe ser capaz de manejar múltiples IDs para un mismo filtro (ej. `dance_style_id=1,2`).
