  • Performance: <500ms authentication time, optimized for edge computing
  • Security: Salt-based password hashing with secure random generation
  • Token Structure: Standard JWT with user context (userId, email, name, roles)
  • Database Schema: Users + Account tables for credential management
  • Production Status: ‚úÖ 100% operational across all environments

    • JWT Validation Architecture

  • Service Binding Delegation: API Gateway delegates token validation to worker-auth
  • Centralized Validation: Single source of truth for JWT verification
  • Cross-Service Communication: Sub-10ms validation via Cloudflare service bindings
  • Error Handling: Comprehensive error responses for invalid/expired tokens
  • Performance: Edge-optimized for minimal latency

    • API Client Implementation

    The frontend connects to the backend API using a typed client with authentication:

  • Backend (`apps/udance-api/src/client.ts`): Exports a Hono client using `hc`
  • Frontend (`apps/udance-web-app/src/lib/client.ts`): Imports and re-exports the client from the API package
  • Authentication: JWT tokens passed via Authorization headers
  • Type Safety: Full type safety including authentication context
  • The client provides type-safe access to all API routes defined in the backend
  • Wrapper functions are created in the frontend for common API operations (e.g., `getTasks()`)
  • Production URLs: Frontend calls production API endpoints in deployed environment
  • Protected Routes: Automatic JWT token inclusion for authenticated requests

    • Backend Architecture

    The backend follows a microservices-like architecture using Cloudflare Workers:

  • Gateway API (`udance-api`): Serves as the entry point and request router with authentication middleware
  • Specialized Workers: Handle specific features (auth, tags, users, etc.)
  • Service Bindings: Enable direct worker-to-worker communication with user context
  • Authentication Middleware: JWT validation via service binding delegation
  • Configuration: Defined in wrangler.toml files for deployment
  • Auth worker (`worker-auth`): SimpleAuth implementation for JWT management
  • Tag worker (`worker-tags`): First specialized worker implemented with authentication protection
