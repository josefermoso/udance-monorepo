udance-monorepo/
├── apps/
│ ├── web/ # Next.js 15 App Principal
│ │ ├── app/ # App Router

│ │ │ └── globals.css # Responsive styles
│ │ ├── components/ # Responsive components
│ │ │ ├── ui/ # Responsive UI kit
│ │ │ ├── layouts/ # Desktop/Mobile layouts
│ │ │ └── mobile/ # Mobile-specific components
│ │ ├── hooks/ # Responsive hooks
│ │ │ ├── useMediaQuery.ts # Breakpoint detection
│ │ │ ├── useTouch.ts # Touch gestures
│ │ │ └── usePWA.ts # PWA features
│ │ ├── next.config.js # PWA + responsive config
│ │ └── package.json
│ │
│ └── storybook/ # Design System Docs
│
├── packages/
│ ├── ui/ # Responsive Design System
│ │ ├── components/ # Multi-viewport components
│ │ │ ├── Button/ # Touch + click optimized
│ │ │ ├── Card/ # Responsive cards
│ │ │ ├── Navigation/ # Desktop/Mobile nav
│ │ │ └── Modal/ # Responsive modals
│ │ └── styles/ # Responsive utilities
│ │ ├── breakpoints.ts # Media queries
│ │ ├── touch.css # Touch optimizations
│ │ └── mobile.css # Mobile-first styles
│ │
│ ├── api-client/ # Supabase Client Wrapper
│ │ ├── auth.ts # Auth functions
│ │ ├── database.ts # DB queries
│ │ ├── realtime.ts # Real-time subs
│ │ └── offline.ts # Offline sync
│ │
│ ├── store/ # Zustand State Management
│ │ ├── auth-store.ts
│ │ ├── ui-store.ts # Responsive UI state
│ │ └── offline-store.ts # PWA offline state
│ │
│ ├── utils/ # Shared utilities
│ │ ├── responsive.ts # Responsive utilities
│ │ ├── touch.ts # Touch utilities
│ │ └── pwa.ts # PWA utilities
│ │
│ └── config/ # Shared config
│ ├── tailwind/ # Responsive Tailwind
│ ├── pwa/ # PWA configs
│ └── typescript/ # TS configs
│
├── turbo.json # Turborepo config
├── package.json # Root dependencies
└── pnpm-workspace.yaml # Workspace config

🔄CI/CD Simplificado

🌊 Development Flow

  • Feature branch → Automatic preview deploy
  • PR opened → CI checks (lint, test, build)
  • Merge main → Auto deploy to staging
  • Manual prod → Deploy to production

🌐 Web App (Vercel)

  • Preview: Cada PR = URL única
  • Staging: Auto-deploy desde main
  • Production: Manual con approval
  • Edge: Global CDN + PWA

📱 PWA Features

  • Auto-Install: Prompt de instalación
  • Offline: Service Worker caching
  • Updates: Background sync automático
  • Performance: Lighthouse optimization

🗄️ Database (Supabase)

  • Migrations: SQL en Git, auto-apply
  • Branching: DB branch per feature
  • Staging: Separate Supabase project
  • Real-time: Sync entre todos los devices
# .github/workflows/deploy.yml name: Deploy Web App on: push: branches: [main] pull_request: branches: [main] jobs: test: runs-on: ubuntu-latest steps: - uses: actions/checkout@v4 - uses: pnpm/action-setup@v2 - run: pnpm install - run: pnpm build - run: pnpm test - run: pnpm lighthouse # PWA audit deploy: needs: test if: github.ref == 'refs/heads/main' runs-on: ubuntu-latest steps: - uses: vercel/action@v1 with: vercel-token: ${{ secrets.VERCEL_TOKEN }}

📱Responsive Components Example

Components que se adaptan automáticamente al viewport:

// packages/ui/components/Navigation/index.tsx import { useMediaQuery } from '../hooks/useMediaQuery' export function Navigation() { const isMobile = useMediaQuery('(max-width: 768px)') return ( <nav className={` ${isMobile ? 'fixed bottom-0 left-0 right-0 bg-white shadow-lg' : 'sidebar w-64 h-screen bg-gray-50' } `}> {isMobile ? <MobileNav /> : <DesktopNav />} </nav> ) } // Responsive hooks export function useTouch() { return { onSwipe: handleSwipe, onPinch: handlePinch, onTap: handleTap } }

Un componente, todos los dispositivos, experiencia optimizada para cada uno.
