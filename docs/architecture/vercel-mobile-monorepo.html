🔄CI/CD Pipeline Moderno

🌊 Development Flow

  • Feature branch → Automatic preview deploy
  • PR opened → CI checks (lint, test, build)
  • Merge main → Auto deploy to staging
  • Manual prod → Deploy to production

🌐 Web App (Vercel)

  • Preview: Cada PR = URL única
  • Staging: Auto-deploy desde main
  • Production: Manual o auto con tags
  • Edge: Global CDN automático

📱 Mobile App (EAS)

  • Development: Expo Go para testing
  • Preview: EAS Update over-the-air
  • Production: App Store + Google Play
  • Hotfixes: OTA updates sin store review

🗄️ Database (Supabase)

  • Migrations: SQL en Git, auto-apply
  • Branching: DB branch per feature
  • Staging: Separate Supabase project
  • Backups: Point-in-time recovery
# .github/workflows/deploy.yml name: Deploy on: push: branches: [main] pull_request: branches: [main] jobs: test: runs-on: ubuntu-latest steps: - uses: actions/checkout@v4 - uses: pnpm/action-setup@v2 - run: pnpm install - run: pnpm build - run: pnpm test deploy-web: needs: test if: github.ref == 'refs/heads/main' runs-on: ubuntu-latest steps: - uses: vercel/action@v1 with: vercel-token: ${{ secrets.VERCEL_TOKEN }} deploy-mobile: needs: test if: github.ref == 'refs/heads/main' runs-on: ubuntu-latest steps: - uses: expo/expo-github-action@v8 - run: eas update --auto

⚡ Beneficios de esta Arquitectura

  • Código Compartido: UI components, business logic, tipos entre web y mobile
  • Desarrollo Paralelo: Web y mobile pueden desarrollarse simultáneamente
  • Type Safety: TypeScript end-to-end desde DB hasta UI
  • Builds Inteligentes: Turborepo solo rebuilds lo que cambió
  • Deployment Automático: Git push → deploy en segundos
  • Hot Reload: Cambios instantáneos en desarrollo
  • Una Base de Datos: Supabase para web y mobile
  • Real-time: Updates instantáneos entre dispositivos

🎨Design System Unificado

Los componentes en packages/ui funcionan tanto en web como mobile:

// packages/ui/components/Button/index.tsx import { Pressable, Text } from 'react-native' import { styled } from 'nativewind' const StyledPressable = styled(Pressable) const StyledText = styled(Text) export function Button({ title, onPress, variant = 'primary' }) { return ( <StyledPressable className={`px-4 py-2 rounded-lg ${ variant === 'primary' ? 'bg-blue-500' : 'bg-gray-300' }`} onPress={onPress} > <StyledText className="text-white font-medium"> {title} </StyledText> </StyledPressable> ) }

Este mismo componente funciona en Next.js y React Native sin cambios.
